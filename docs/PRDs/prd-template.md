# PRD Template

## Product Requirements Document (PRD)

### 0. Revision History

| Date       | Author | Change Description |
| ---------- | ------ | ------------------ |
| YYYY-MM-DD | Name   | Initial draft      |

### 1. Overview

- **Problem Statement:** What problem are we solving?
- **Value Proposition:** Why does this matter? What is the benefit?

### 2. Goals & Objectives

**Goals:**

- List the high-level, strategic goals for the product (e.g., solve a specific user problem, enter a new market).

**Objectives:**

- List the measurable, actionable objectives that support the goals (e.g., achieve X% adoption in 6 months, reduce onboarding time by Y%).

### 3. Stakeholders

| Name/Group             | Role/Responsibility | Contact/Notes       |
| ---------------------- | ------------------- | ------------------- |
| Example: Product Owner | Decision-maker      | `email@example.com` |
| Example: Engineering   | Implementation Lead |                     |
| Example: QA            | Reviewer/Tester     |                     |

List all main stakeholders, their roles, and relevant contact or notes. Add/remove rows as needed.

### 4. Specifications & Use Cases

- Provide clear specifications for product features and behaviors.
- Use cases may be included to illustrate intended usage, but are optional.
- If user stories are provided, include acceptance criteria for clarity.

### 5. Functional Requirements

- List core features and functionalities required for the product.
- Assign a priority (Must/Should/Could) to each requirement.

### 6. Out of Scope

- Explicitly state what will NOT be delivered in this release.

### 7. Non-Functional Requirements

- Specify performance, security, scalability, usability, and other non-functional criteria.

### 8. Success Metrics

- Define how success will be measured (KPIs, adoption, satisfaction, etc.).
- Link metrics to business outcomes.

### 9. Constraints & Assumptions

- Document any known constraints, dependencies, or assumptions.

### 10. Timeline & Milestones

- Outline the expected timeline, major milestones, and delivery phases.
- List dependencies and note critical path milestones.

### 11. Risks & Mitigations

- List potential risks and proposed mitigation strategies.

### 12. Open Questions

- Document unresolved issues or decisions.

### 13. References & Related Documents

- Link to wireframes, mockups, technical specs, research, or other supporting docs.
